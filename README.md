# REF Compliance Report
A R/shiny app that checks a Figshare batch download for REF compliance

Build file:
1.	Batch download all public items from repo
2.  Excludes items pre-2021
3.	Include only items that have an embargo date set in the future / permanent embargo
4.	Include only journal articles, chapters, and conference papers with ISSNs

Generate reports:

Report 1: Permanent embargo report: 
  - (Assumed that these items do not have the correct version of the file)
  - Includes all items without an embargo end date
  - Excludes items posted 3+ months after publication
  - Items that are within 0-1 months of publication, flagged as RED
  - Items that are within 2-3 months of publication, flagged as AMBER
  - Items that are 3+ months of publication, flagged as GREY
  - Items with no publication date, flagged as GREEN
  - User can add in status & comment
    
Report 2: Change report
  - (Assumes that version of file has been corrected)
  - Generated by uploading a previous permanent embargo report
  - Lists items that were on permanent embargo report, but now are either: (1) unpublished from the Repository, (2) on temporary embargo, (3) no embargo
  - Will include columns for status & comment pulled from previous permanent embargo report
  
Report 3: Temporary report
  - Lists all items with a temporary embargo
  - Includes timeline column, difference between embargo date & published date
    - No publication date GREEN
    - Flagged RED
      - Pre-2026, STEM, 12+ months
      - Pre-2026, HASS, 24+ months
      - 2026+ STEM, 6+ months
      - 2026+ HASS, 12+ months

Sorted by embargo date
Report columns:

  - Item_id
  - Author
  - Title
  - Item type
  - Handle
  - Posted date
  - Embargo date
  - Acceptance date
  - Publication date
  - Version
  - Flag
  - Status [blank on first report]
  - Comment [blank on first report]
